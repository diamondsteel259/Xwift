## Technical Deep Dive

### For Technical Audiences

**Network Architecture:**
- P2P network with flood-based propagation
- Dandelion++ for transaction privacy
- DHT-based peer discovery
- Tor/I2P integration supported

**Consensus Mechanism:**
- Proof of Work (PoW)
- RandomX algorithm (CPU-optimized)
- Difficulty adjustment every 8 blocks (4 minutes)
- Same security model as Monero

**Privacy Technology:**
- **Ring Signatures:** CLSAG (Concise Linkable Spontaneous Anonymous Group)
- **Stealth Addresses:** Diffie-Hellman key exchange with one-time keys
- **RingCT:** Pedersen commitments with Bulletproof+ range proofs
- **Ring Size:** 16 (minimum, configurable)
- **View Tags:** Enabled for wallet scanning optimization

**Transaction Structure:**
- Inputs: Ring signatures with key images
- Outputs: Stealth addresses with encrypted amounts
- Range Proofs: Bulletproof+ (compact and efficient)
- Signature: CLSAG for entire transaction

**Block Structure:**
- Block version: Compatible with Monero v15+
- Coinbase transaction: Miner reward + dev fund output (year 1)
- Transaction tree: Merkle tree of transaction hashes
- Difficulty: Stored in block header

**Emission Curve:**
```
reward = (M - A) * 2^-20 * 10^-12 + tail
where:
  M = MONEY_SUPPLY (effective ~108.8M XFT)
```
